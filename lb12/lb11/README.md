Лабораторная работа №12: Разработка SPA с использованием Pinia

Описание приложения
Приложение представляет собой трёхстраничный интернет-магазин (SPA), разработанный на фреймворке Vue 3 с использованием Composition API (`<script setup>`). 

Основная задача приложения — демонстрация перехода от локального управления данными внутри компонентов к **централизованному управлению состоянием** с помощью библиотеки Pinia. Все переменные страниц (состояние полей формы, массивы данных) и логика сетевых запросов (`fetch`) полностью вынесены из компонентов в единое глобальное хранилище (Data Store), что устраняет проблему избыточной передачи пропсов (Props Drilling).

Структура приложения 
1. Главная страница (`/`) — отображает статичную текстовую информацию о преимуществах магазина (ассортимент, доставка, поддержка).
2. Каталог товаров (`/products`) — при открытии страницы автоматически вызывает действие из хранилища Pinia, выполняет асинхронный GET-запрос к внешнему API (`fakestoreapi.com`) и динамически выводит сетку из 6 карточек товаров.
3. Контакты (`/contact`) — содержит форму обратной связи. Данные из инпутов синхронизируются со стейтом Pinia через двустороннее связывание (`v-model`). При клике на кнопку вызывается метод отправки из хранилища, который отправляет POST-запрос в формате JSON на тестовый сервер `jsonplaceholder.typicode.com` и выводит статус ответа.

Команды для сборки и запуска

```bash
# Установка всех зависимостей проекта (Vue, Vue Router, Pinia, Bootstrap)
npm install

# Запуск локального сервера разработки (по адресу http://localhost:5173/)
npm run dev
